Custom classes

You probably already know: object helps you keeping the code clean and readable, easy to modify and easy to understand.
As this weren't enought Gishiki also includes an ORM, that allows you to store, restore, udate and delete objects on databases.

For those reasons you might want to import classes on your application. The most obvious way to do this is doing: include("myClass.php"); in each controller.
This is really bad: what about if you need to add another class or delete one? You have to modify each controller that includes the modified class, and you have to remember them.
To avoid this classes inside the directory "Classes", inside the "application" directory are automatically included.


inclusion directory: Only files inside the "Classes" directory are autometically included. No files from subdirectories are included.
You can change the classes inclusion directory editing the config.php file.


inclusion check: After the inclusion of a file, a check is performed: if the included file doesn't contains a class with the name of the included file an error is shown.
This is for helping you maintaning a good class management.

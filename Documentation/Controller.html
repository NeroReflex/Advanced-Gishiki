Controllers

A controller is represented as a class with functions needed to manipulate data through a model and render the final content through one or more views, may that contant be an HTML page or a JSON object.
The task of creating a controller is really easy and takes no time at all.
To create controllers you must keep in your mind three things:
  • The class must be named as the controller + _Controller, and must be placed in a PHP file with the name of the controller, inside the controller directory
  • You have to create a new class that inherits from the Gishiki_Controller class, which is the base controller class
  • If you are using a custom class contructor, you have to remember to do parent::__construct(); in the first line of your controller class constructor
  • If you are using a custom class destructor, you have to remember to do parent::__destruct(); in the first line of your controller class destructor

You can see an example above: 
class My_controller_Controller extends Gishiki_Controller {
    function __construct($arguments)
    {
	    parent::__construct($arguments);

        /*   TODO: add your code here   */
    }

    function __destruct()
    {
	    parent::__destruct();

        /*   TODO: add your code here   */
    }
}
Remember to save this file as My_controller.php inside the controller directory and you have created your new controller.

However this is not enough to create a working controller. You have to create, for each controller, at least the Index function. The Index function is the default action for a controller, and is called when no specific actions are required by the browser/client.

For example think about a browser that request the page yourdomain.com/My_controller. You know that the controller that must be used is your new controller, but you can't figure out what is the action to be performed.
Index is that default action! For Gishiki a controller that doesn't have an Index function and/or doesn't inherit from Gishiki_Controller is not a valid controller.

A controller instance is created automatically by Gishiki, and the given function is executed automatically. In order to have a function to be executed the browser must be redirected to: yourdomain.com/Controller/Function. This will have Gishiki execute the Function function inside the new object instantiated from the Controller class.

Keep in mind that if no controller is given, than the Default controller is executed. This means you should include a controller called Default_Controller.

That said, a minimum valid controller is: 
class My_controller_Controller extends Gishiki_Controller {
    function __construct($arguments)
    {
	    parent::__construct($arguments);

        /*   TODO: add your code here   */
    }

    function Index()
    {
        /*   TODO: add your code here to do stuff   */
    }

    function __destruct()
    {
	    parent::__destruct();

        /*   TODO: add your code here   */
    }
}
Remember to save this controller with the name: My_controller.php in order to get it working.
Tip: you are not obliged to define a class constructor or destructor.
you don't need a class constructor or destructor, just use the default one: 
class My_controller_Controller extends Gishiki_Controller {
    
    function Index()
    {
        /*   TODO: add your code here to do stuff   */
    }
}
This still is a valid controller.

Model within a controller

How to create a Model is discussed in the model chapter. This section shows you how to use a model. For each instantiated controller a model is automatically joined. This means that a model instance is every time available within your controller.
The proper way to access your model in order to perform an operation is using the the ExecuteOperation function, exposed by the model manager itself.

An example of model usage is shown below: 
class My_controller_Controller extends Gishiki_Controller {
    function __construct($arguments)
    {
	    parent::__construct($arguments);

        /*   TODO: add your code here   */
    }

    function Index()
    {
        //get the list of all users
        $users = $this->Model->ExecuteOperation("getUserList");
    }
}
This example shows you how to call an operation defined in a model that returns somethig. Please, note that you are the one who names operations, and write them, and than you are the one who know what they return.

However sometimes you need to pass data to the model to perform the requested operation. This is done by passing an array to the operation, that receives it. This will be described later, just keep you mind that it is possible to do things such as: 
class My_controller_Controller extends Gishiki_Controller {
    function __construct($arguments)
    {
	    parent::__construct($arguments);

        /*   TODO: add your code here   */
    }

    function Index()
    {
        $agedUsers = array(
            "maxAge" => 18,
            "minAge" => 21,
        );

        //get the list of users that are 18, 19, 20 or 21 years old
        $users = $this->Model->ExecuteOperation("getAgedUserList", $agedUsers);
    }
}
Sounds familiar, doesn't it?

In the documentation chapter about Models you will find more complete examples on how you create and use models.

Digital Signature with Asymmetric cipher

The digital signature feature uses the AsymmetricCipher class, and is part of it.

The digital signature feature is internally used by the framework to check if cookies were edited one the client-side.

In order to use the digital signature feature it is required to instantiate an AsymmetricCipher object.
After that it is necessary to load a key, using LoadKey, or create a new random key using GenerateNewKey to generate a new one.

So, to digitally sign a message, or to check a message with its signature, you do things exactly like as you would do if you were going to encrypt data with an asymmetric cipher algorithm:

class Crypto_Controller extends Gishiki_Controller {
    function RSAEncryptionTest()
    {
        //get the string to sign from an url like http://localhost/CryptoRSA/RSAEncryptionTest/....................
        $encrypted = $receivedDetails[0];

        //setup the symmetric cipher
        $asymmCipher = new AsymmetricCipher();

        //change the default key with the given one
        $asymmCipher->GenerateNewKey("testKey", AsymmetricCipher::RSA4096);

        //crypt and decrypt data
        $signature = $asymmCipher->GenerateDigitalSignature($dataToSign);
        $signatureCheckingResult = $asymmCipher->VerifyDigitalSignature($dataToSign, $signature);

        $signatureCheckingStringResult = "data is incorrect";
        if ($signatureCheckingResult) {
            $signatureCheckingStringResult = "data is correct";
        }

        //prepare the page to be shown
        $dataToDisplay = array(
            "sign" => $signature,
            "signRes" => $signatureCheckingStringResult,
        );
        
        //show the page
        ProcessPartialView("RSAEncryptionTest", $dataToDisplay);
    }
}
Save this page as CryptoRSA.php, create a CryptoRSA.php that contains an empty model and create an RSAEncryptionTest.html like this one: 
<!DOCTYPE html>

<html lang="en">
    <head>
        <meta charset="utf-8" />
        <title>RSA Encryption test</title>
    </head>
    <body>
        The digital sign of the message is:<br />
        {{sign}}
        <br /><br /><br />
        The check result result is:<br />
        {{signRes}}
    </body>
</html>
Direct the browser to http://localhost/CryptoRSA/RSAEncryptionTest/This+is+the+message+that+was+encrypted+%3B-) As you can see the digital sign (the string returned by the GenerateDigitalSignature function) is base64 encoded, because, once encoded it can be displayed completely by the browser/client. Keep in mind that the result il base64 encoded when you develop your custom client.

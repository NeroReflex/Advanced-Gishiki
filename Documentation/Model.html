Models

A model is, basically, like a controller, a class. Creating a model is easy as creating a controller, you just need to keep in mind that:
  • The class must be named as the model + _Model, and must be placed in a PHP file with the name of the model, inside the model directory
  • You have to create a new class that inherits from the Gishiki_Model class, which is the base model class
  • If you are using a custom class contructor, you have to remember to do parent::__construct(); in the first line of your model class constructor

This is an example of a model: 
class User_Model extends Gishiki_Model {
    function __construct()
    {
	    parent::__construct();

        /*   TODO: add your code here   */
    }

    public function __destruct() {
	    parent::__destruct();

        /*   TODO: add your code here   */
    }
}
Remember to save this file as User.php inside the model directory and you have created your new controller.

Tip: Remember that you are not obliged to define a class constructor. If you don't need it, just use the default one.

The model is where you load and store data from a data collection. A data collection can be any type of data collection, for example a database (SQLite, MySQL or MSSQL), a JSON file or a simple text file.

Despite the fact you can store information where you want, Gishiki gives you additional power helping you to store and load data from a RDBMS.

On the controller chapter there are two examples that shows you to perform a model operation within a controller: 
$users = $userModel->ExecuteOperation("getAgedUserList", $agedUsers);
When you read that you may have asked: Where and what getAgedUserList is? Well, getAgedUserList is a function inside your model. That function accept an array and return an array of fetched data. 
class User_Model extends Gishiki_Model {
    function getAgedUserList($usersSpecifications = null)
    {
        //setup the array that will be returned
        $users = array();

        /*  use $usersSpecifications to fetch $users */

        //and return that array
        return $users;
    }
}
You have already seen somethig similar, don't you :D ?

Remember that: you are the one who decide what your function returns, but I am the one who has decided that your function will take only an argument (I suggest you to use an array, but you can use anything).

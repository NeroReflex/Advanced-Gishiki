Selection criteria

A single clause is composed by a field name, a field value and the criteria that specify how to select objects to be restored or deleted.

You can find each possible criteria inside the Criteria abstract class:
  • Criteria::EQUAL
  • Criteria::NOT_EQUAL
  • Criteria::GREATER_THAN
  • Criteria::LESS_THAN
  • Criteria::GREATER_EQUAL
  • Criteria::LESS_EQUAL
  • Criteria::LIKE
  • Criteria::NOT_LIKE
  • Criteria::IS_NULL
  • Criteria::IS_NOT_NULL
Criteria::LIKE and Criteria::NOT_LIKE are reserved to strings-only values.
Criteria::EQUAL and Criteria::NOT_EQUAL are both valid for every data types.
Any other criteria is reserved to numbers, both integers and decimals.
When usingCriteria::IS_NULL and Criteria::IS_NOT_NULL you don't need to specify the value for that clause.

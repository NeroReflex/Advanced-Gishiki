Symmetric cipher

Gishiki has its own symmetric cipher implementation, based on the OpenSSL PHP extension. OpenSSL is really important in order to run this framework, because it is used for everything that concerns security.
OpenSSL was chosen because it is released in the PHP bundle, it is the fastest library for cryptography and because it has all the necessary algorithms.

The symmetric cipher is used by the framework to encrypt the password in a database connection using the default password and the AES192 algorithm.

In order to use the symmetric cipher it is required to instantiate a SymmetricCipher object.
If no key is given (using SetKey) than the default one is used.

Using the OpenSSL program it is possible to decrypt a message encrypted by the symmetric cipher, but it is necessary that you understand how the symmetric cipher encrypts your messages.

Let's try to encrypt something: 
class Crypto_Controller extends Gishiki_Controller {
    function EncryptionTest()
    {
        //get the string http://localhost/Crypto/EncryptionTest/....................
        $dataToEncrypt = $receivedDetails[0];

        //setup the symmetric cipher
        $symmCipher = new SymmetricCipher(SymmetricCipher::AES256);

        //change the default key with the given one
        $symmCipher->SetKey("This is the key used to encrypt AND decrypt");

        //crypt and decrypt data
        $encrypted = $symmCipher->Encrypt($dataToEncrypt);
        $decrypted = $symmCipher->Decrypt($encrypted);

        //prepare the page to be shown
        $dataToDisplay = array(
            "enc" => $encrypted,
            "dec" => $decrypted,
        );
        
        //show the page
        $this->ProcessPartialView("encryptionTest", $dataToDisplay);
    }
}
Save this page as Crypto.php, create a Crypto.php that contains an empty model and create an encryptionTest.html like this one: 
<!DOCTYPE html>

<html lang="en">
    <head>
        <meta charset="utf-8" />
        <title>AES256 Encryption test</title>
    </head>
    <body>
        The encryption result is:<br />
        {{enc}}
        <br /><br /><br />
        The deryption result is:<br />
        {{dec}}
    </body>
</html>
Direct the browser to http://localhost/Crypto/EncryptionTest/This+is+the+message+that+was+encrypted+%3B-) As you can see the encryption result (the string returned by the Encrypt function) is base64 encoded, because, once encoded it can be displayed completely by the browser/client.

Note that: it is possible to use the OpenSSL library (or one of its porting) to write a program that decrypts and encrypts messages with the same algorithm used by Gishiki, but no sources will be given to accomplish this job.

Gishiki

The Gishiki framework follows the MVC pattern to display content to a client. That content can be an HTML page or a JSON object, that will be used with javascript or other applications.

The MVC pattern is based around three components: Model, View and Controller.

The controller is used to elaborate a request from the client and the given data, in order to have the model retrive or store that data. The last task that a controller perform is creating the final result using one or more Views. You can see the controller as a glue for models and views.

The model is called by the controller and is used to store or retrive data from a data collection. The default data collection in Gishiki is a SQLite3 database, but you can use everything you want: from MySQL to a text file, or even JSON. The choice is yours.

The view is the model of the output data. You define how your data is going to be organized and Gishiki place the data in that model.

This pattern is used to separate logic, data storing operations and display operations.
